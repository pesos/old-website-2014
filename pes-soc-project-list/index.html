Universal Gestural Remote
Concept: The idea is to build a gesture-based device that can be used to control many different things, just by pointing and gesturing at the object you want to control. This could be used to build an interactive (and much clichéd) home automation system. Think of turning on the light just by pointing and clicking at it, or controlling the curtains by pointing at it and making a rolling gesture by slightly rotating your hand. You could control your coffee table and make it come to you!
Goals:The goals of this project are to develop an extremely low-cost, as well as a robust and scalable solution for the proposed concept. The design consists of 2 parts: design of the gesturebased device, and the design of the object(s) to be controlled. The objects to be controlled must not have very complicated controls. You can’t fly an airplane or control a real life car with this, but something like lights, fans, curtains, computer mice, which have a small set of control inputs. By the end of 2 months, we expect to build a complete system prototype that can demonstrate this concept. Proposed Design Architecture:
Gesture device: A accelerometer tilt based glove (if possible, watch/wristband). It would have a small infra-red LED pointing in the direction of the stretched hand, working like a TV remote.
Controlled object: Must have a IR LED (TSOP1738)sensor which would read the gesture instruction and act accordingly.
Example: You point at a light and turn your hand in one direction. The LED on your hand sends a code that corresponds to that action. The lighting unit has a sensor which reads this instruction and looks at it mapping table. It finds that that instruction corresponds to turning on the light, so it does that. The light turns on. Note: This design is subject to changes. It is only the tentatively proposed design and can change if needed.
Requirements: We will require 2-3 interns for this project. They must be comfortable in working with analog hardware and in programming an Arduino. They must have basic familiarity with C. Python is an added plus, but not really required.
Design Challenges (as of now):

  1. Pointing accuracy vs spatial distribution of the objects to the controlled. What is the most optimal?
  2. Usability of gesture device, multiplexing different instructions. What are the limits, what would be the most user-friendly?
  3. Do we need to encrypt signals so that only specific people can operate on specific objects? How would that work?
and many more.

Project Name: PES Code Fest


Project Description: An online coding competition exclusively for PES students. The admins will put up a set of questions for the students to solve using clever algorithms implemented in programming language of their choice.
Implementation Details
Admins will have the option of scheduling the contest, number of rounds in the competition, the questions in each round, point weightage for each question, time limit for each round, upload input files and corresponding output files also.
Students should register to this web app, n solve the problems by downloading the input file and feeding it to their programs (at their own end system) and uploading the resulting output and the program file. The timer starts when the user downloads the input file and expires if the output is not uploaded within the given time(to test the time efficiency of the algorithm).
The application should check for the validity and correctness of the output uploaded by the user.
Aim of the project:
  • To help students reach the next level of thinking.
  • A better understanding of existing algorithms and persuade them to design new ones.
  • Helps students solve challenging questions in technical rounds of placement interviews.
Feature addition or enhancement are welcome.
Prerequisites: (Learn while you do)
  • HTML5, CSS, Javascript for front-end
  • PHP,Python(django),NodeJS or ROR and MySQL for back-end
Mentor: Vinod Kumar L: vinodkumarlogan AT gmail DOT com

COllaborative Placement PReparation


Placements play a very important role in every student's academic life. Placement examinations and interviews test a student to the very maximum extent and determine whether he/she is ready to face the challenges in the real world.
A lot of studying is needed to crack screening tests and interviews. Four years' worth of knowledge needs to be revised in just a few weeks, which is not a trivial task. This daunting task is easier if done collaboratively.
COPPR is a platform for sharing materials and resources. Notes, links, questions, experiences can be shared and searched against so that everyone can help everyone else and can expect the same in return.
COPPR is to be implemented as a web application as this simplifies matters when it comes to sharing and syncing information. The syncing of information is expected to be real-time as a lot of the placement activities happen in a short duration (~2 weeks) and every student aiming to get placed is studying all the time on the day of the tests.
Prerequisites:
  • Knowledge of programming in atleast one language (Javascript preferred)
  • Knowledge (at least basics) of web programming (HTML, CSS, JS)
  • Knowledge of any one DBMS
